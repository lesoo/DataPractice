혼동행렬(Confusion Matrix)¶

자주 사용하는 분류 문제의 평가지표¶

  1. accuracy: 전체 중에서 맞춘 비율
  • acc = (TP + TN) / (TP + TN + FN + FP)
  1. precision: T 라고 분류한 것 중에서 실제 T인 것의 비율
  • prec = TP / (TP + FP)
  1. recall: 실제 T 인 것 중에서 모델이 T라고 예측한 것의 비율
  • recall = TP / (TP + FN)
  • ex) 실제 날씨가 맑은 날 중에서 모델이 맑다고 예측한 비율을 나타낸 지표
  • sensitivity, hit rate
  1. fall-out: 실제 F인 데이터 중에서 모델이 T라고 예측한 비율
  • fall = FP / (FP + TN)

언제 accuracy 대신 prec 과 recall 등이 고려될까?¶

  • 클래스 불균형 문제가 발생할 때 acc의 의미가 떨어짐

prec, recall의 차이¶

  • ex) 화재경보(불이 남: 1, 안 남: 0)
  • FP는 불 났다고 판단했으나 불 안남, FN는 불 안 났다고 판단했으나 불 남

  • 경중의 차이가 큼 -> FN에 중점을 둬야함

  • ex) 스팸메일 분류(스팸: 1, 스팸 x: 0)

  • FP는 스팸이라 판단했으나 스팸 아님, FN는 스팸이 아니라 판단했으나 스팸임

  • 중요한 메일이 오기로 했는데 스팸으로 분류되면 확인 못함 -> FP

  • ex) 환자 검진(병: 1, 병x: 0)

  • ex) 금융 사기 탐지(사기: 1, 아니: 0)

  • ex) 게임 어뷰저 분류(어뷰저: 1, 아니: 0)

ROC curve는 어떻게 그려질까?¶

  • 확신의 정도와 실제 정답을 이용해 y축을 recall, x축을 fall-out 좌표로 이용
  • 전부 true라 예측하면? -> recall: 1.0, fallout: 1.0 -> (1, 1) 좌표
  • 전부 false라 예측하면? -> recall: 0.0, fallout: 0.0 -> (0, 0) 좌표
  • 판단 기준(threshold)을 0~100%까지 조금씩 바꾸면서 그때마다 혼동행렬 획득 -> 평가지표 값을 산출
  • ex) 판단 기준을 0, 10, 20, .... 100%으로 총 11번 다르게 한다면? -> 혼동행렬도 11개 나오고, 좌표 평면에 점을 11개 찍을 수 있음 -> 이 점들을 이어 그리면 ROC_Curve 완성

def scorer(model):
    model.fit(X_train, y_train)
    for_plot = pd.DataFrame()
    for_plot['predict'] = model.predict(X_test)
    for_plot['actual'] = y_test

    TN, FP, FN, TP = cm(for_plot['actual'], for_plot['predict']).ravel()
    recall = TP / (TP + FN)
    return recall

md_ls = [3, 5, 7, 9, 11, 13, 15, 17]
mf_ls = [0.1, 0.3, 0.5, 0.7, 0.9]
all_scores = []

for md in md_ls:
    for mf in mf_ls:
        model = rf(max_depth=md, max_features=mf, n_jobs=-1)
        all_scores.append({'recall': scorer(model), 'md':md, 'mf':mf})

hyper_df = pd.DataFrame(all_scores)
hyper_df.sort_values(by='recall', ascending=False)
